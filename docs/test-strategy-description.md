# OFI Mean Reversion Strategy

Стратегия для HFT-бэктестера на tape data (сделки без стакана).  
Сигнал основан на **Order Flow Imbalance** — дисбалансе объёма покупок и продаж в скользящем окне.

---

## Идея

На коротких горизонтах агрессивные покупатели давят цену вверх. Если за последние N миллисекунд доля buy-объёма превышает порог, с высокой вероятностью цена продолжит рост — входим в лонг и фиксируем прибыль на небольшом движении.

---

## Сигнал: Order Flow Imbalance

$$\text{OFI} = \frac{V_{buy} - V_{sell}}{V_{buy} + V_{sell}} \in [-1, +1]$$

Скользящее окно `window_us` (по умолчанию 500 мс) содержит все сделки за последний период.  
Вход в лонг, когда `OFI ≥ imbalance_threshold` (по умолчанию 0.65).

---

## Параметры

| Параметр | По умолчанию | Описание |
|---|---|---|
| `window_us` | 500 000 мкс | Ширина окна для расчёта OFI |
| `imbalance_threshold` | 0.65 | Минимальный OFI для входа |
| `take_profit_bps` | 15 bps | Целевая прибыль (+0.15%) |
| `stop_loss_bps` | 8 bps | Максимальный убыток (−0.08%) |
| `order_quantity` | 100 | Размер позиции в лотах |
| `commission_bps` | 10 bps | Комиссия на сторону |
| `ORDER_TIMEOUT_US` | 500 000 мкс | Время жизни лимитного ордера |
| `LIMIT_PRICE_BUFFER_BPS` | 5 bps | Буфер сверху рынка для лимитки |

---

## Логика исполнения

```
MarketEvent
    │
    ├─ updateOFI()          — добавить тик в окно, удалить устаревшие
    │
    ├─ checkPendingOrders() — проверить исполнение / отмену лимитного ордера
    │
    ├─ position == 0 ?
    │       └─ tryEnter()   — OFI ≥ threshold → лимитный buy (+5 bps к цене)
    │
    └─ position > 0 ?
            └─ tryExit()    — цена +15 bps → take profit
                           — цена  −8 bps → stop loss
```

### Вход
Выставляется один лимитный ордер на `order_quantity` лотов по цене `last_price × (1 + 5 bps)`.  
Если ордер не исполнился за 500 мс — отменяется. Новый ордер не выставляется, пока есть активный.

### Выход
Рыночный ордер при достижении take profit или stop loss.  
PnL рассчитывается с учётом комиссии на обе стороны сделки.

## Метрики в `on_finish()`

| Метрика | Описание |
|---|---|
| `Trades` | Общее число сделок (вход + выход = 2 на круг) |
| `Win rate` | Доля прибыльных закрытий |
| `Buy signals` | Сколько раз сработал OFI-сигнал |
| `PnL (USD)` | Суммарный реализованный PnL с учётом комиссии |
| `Max Drawdown` | Максимальная просадка от пика капитала |
| `Final Cash` | Остаток кэша на конец прогона |
