# HFT Backtester — Technical Assignment №1

**CMF School | Advanced Quantitative Analytics & HFT Program**

A high-performance C++ backtesting engine implementing Avellaneda-Stoikov (2008) market making strategy with Microprice enhancement (Stoikov 2018).

## Build Instructions

### Prerequisites

- CMake >= 3.20
- GCC >= 11 or Clang >= 13
- Ninja
- Git (for FetchContent)
- Python 3 + numpy + pandas (for microprice computation)

### Windows (MSYS2/MinGW)

```powershell
# Add to PATH
$env:PATH = "C:\msys64\mingw64\bin;C:\msys64\usr\bin;" + $env:PATH

# Configure and build
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build -j4
```

### Linux/macOS

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j4
```

## Running the Strategy Backtester

### Step 1 — Compute Full Microprice Table

```powershell
py scripts/compute_microprice.py data/lob.csv data/microprice_table.csv
```

This estimates the G* adjustment table from historical LOB data (~2 minutes).

### Step 2 — Run Simulation

```powershell
.\build\bin\strategy_backtester.exe
```

Runs all three strategies and prints comparison:
- Avellaneda-Stoikov (2008)
- Weighted Microprice + AS (2018)
- Full Microprice + AS (Stoikov 2018)

### Step 3 — Run Tests

```powershell
.\build\bin\tests.exe
```

## Configuration

Edit `configs/as_config.json` to change strategy parameters (no recompilation needed):

```json
{
    "gamma":       0.01,    // Risk aversion [0.001 - 0.5]
    "kappa":       1.5,     // Order flow intensity [0.5 - 5.0]
    "T":           1.0,     // Trading horizon in seconds
    "q_max":       10000000,// Maximum inventory position
    "order_size":  50000,   // Size of each limit order
    "vol_window":  50       // Snapshots for volatility estimation
}
```

**Parameter guide:**
- Higher `gamma` → tighter inventory management, narrower quotes
- Higher `kappa` → tighter spread, more fills
- Higher `order_size` → larger trades, more turnover

## Execution Model

**Execution assumption** (per assignment specification):

> Execution occurs when the market price crosses the order level.

Implementation in `OrderManager.hpp`:
- **Bid fills** when a SELL trade arrives at price ≤ our bid
- **Ask fills** when a BUY trade arrives at price ≥ our ask

Partial fills are not implemented (optional per specification).
